interface. These messages allow you to track everything the agent does in
real-time, from tool invocations to background tasks to API errors.

KEY FINDING: HOW TO DETECT INTERMEDIATE ACTIONS
================================================================================

1. TOOL INVOCATIONS
   WHERE: SDKAssistantMessage.message.content[].tool_use
   WHAT: Every tool the agent decides to run
   EXAMPLE:
   {
     type: 'tool_use',
     id: 'call_1',
     name: 'Bash',
     input: { command: 'find . -name "*.ts"' }
   }

2. TOOL PROGRESS
   WHERE: SDKToolProgressMessage
   WHAT: Periodic updates while tool executes (every few seconds)
   EXAMPLE:
   {
     type: 'tool_progress',
     tool_name: 'Bash',
     tool_use_id: 'call_1',
     elapsed_time_seconds: 2.3
   }

3. TOOL SUMMARIES
   WHERE: SDKToolUseSummaryMessage
   WHAT: Natural language summary of tool batch completion
   EXAMPLE:
   {
     type: 'tool_use_summary',
     summary: 'Executed 2 bash commands to find files',
     preceding_tool_use_ids: ['call_1', 'call_2']
   }

4. BACKGROUND TASKS
   WHERE: SDKTaskStartedMessage → SDKTaskProgressMessage → SDKTaskNotificationMessage
   WHAT: Multi-step background operations
   
5. SESSION STATE
   WHERE: SDKSessionStateChangedMessage
   WHAT: idle/running/requires_action transitions

6. ERROR CONTEXT
   WHERE: SDKAPIRetryMessage
   WHAT: When and why API calls fail
